JSON Server API Guidelines

This document outlines guidelines for interacting with the JSON Server API, which consists of the following endpoints:

  • https://brazil.harrispye.com/mock/posts: Contains 100 items.
  • https://brazil.harrispye.com/mock/comments: Contains 500 items.
  • https://brazil.harrispye.com/mock/users: Contains 10 items.

General Guidelines

  • All requests must be sent as application/json.
  • Responses will be in JSON format.
  • Ensure proper authentication and authorization mechanisms are in place if necessary.

Endpoints

GET /posts

  • Retrieves a list of posts.
  • Additional query parameters can be used for filtering, sorting, and pagination.

Request:

GET /posts

Response:

[
  {
    "userId": 1,
    "id": 1,
    "title": "sunt aut facere repellat provident occaecati excepturi optio reprehenderit",
    "body": "quia et suscipit\nsuscipit recusandae consequuntur expedita et cum\nreprehenderit molestiae ut ut quas totam\nnostrum rerum est autem sunt rem eveniet architecto"
  },
  // More posts
]

POST /posts

  • Creates a new post.
  • Use the request body to specify the post data.

Request:

POST /posts
Content-Type: application/json
{
  "title": "New Post",
  "body": "This is a new post.",
  "userId": 1
}

Response:

{
  "id": 101,
  "title": "New Post",
  "body": "This is a new post.",
  "userId": 1
  // Other post properties
}

PUT /posts/:id

  • Updates an existing post.
  • Specify the post ID in the endpoint URL and provide the updated data in the request body.

Request:

PUT /posts/1
Content-Type: application/json

{
  "title": "Updated Post",
  "body": "This is an updated post."
}

Response:

{
  "id": 1,
  "title": "Updated Post",
  "body": "This is an updated post.",
  "userId": 1
  // Other post properties
}

DELETE /posts/:id

  • Deletes a post with the specified ID.

Request:

DELETE /posts/1

Response:

{}

Similar Guidelines for /comments and /users

Follow similar request and response patterns for the /comments and /users endpoints. Use the appropriate query parameters and request body structures as needed.

Refer to the JSON Server documentation for more options and advanced usage.
